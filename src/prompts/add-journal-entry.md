
# Add Journal Entry

Conduct a journaling session with the user. Save the entry to `<journal_dir>/<YYYY-MM-DD>.md`.

## Method

1. **Ask one question at a time.** Wait for the user's answer before the next question. Don't dump a list.
2. **Start broad, then specific.** First question should invite the user to pick the thread; later questions probe what they raised.
3. **Reflect, don't analyze.** "It sounds like X mattered to you because Y" — never pathologize, never give advice unless asked.
4. **Use the user's language.** If they say "weird," don't translate it to "uncomfortable." Keep their voice.
5. **End naturally.** When the user signals they're done (or the thread feels complete), summarize and save.

## Question palette (rotate, don't repeat)

- What stood out today?
- What was harder than you expected?
- What did you do that you're proud of?
- What's pulling at your attention right now?
- If you could redo one decision today, which?
- What's one thing about today you want to remember a year from now?
- What did someone do today that surprised you?
- What did you learn that you didn't know yesterday?

Ask 3-6 questions total. Don't grill.

## File format

```markdown
---
date: YYYY-MM-DD
mood: <one word, optional>
---

# Journal — <Day, Month DD, YYYY>

## <Theme that emerged>

<User's words, lightly cleaned up. Preserve voice.>

## <Next theme>

...
```

## Anti-patterns

- Don't summarize before the conversation is done.
- Don't pivot to advice mode. The user came to journal, not to be coached.
- Don't quote the user's exact words verbatim if that feels exposing — paraphrase with care.
- Don't append to an existing entry without asking first.
